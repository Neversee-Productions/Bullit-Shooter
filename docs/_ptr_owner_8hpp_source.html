26 // Some notes on derived-to-base conversion and why an additional indirection object (PtrIndirection) is necessary:
27 // PtrOwnerBase must be a template of T. If it were not (like the reference counter in std::shared_ptr), then there would be no way to extract the
28 // T* pointer of a PtrOwner [1], which is required after copying the PtrOwner. However, as long as PtrOwnerBase<T> is a template, no two CopiedPtr<X>
29 // with different X can refer to the same type-erased base class template PtrOwnerBase. Thus, the PtrOwner implementation would have to create a
30 // PtrOwner<U> with U != T, and because PtrOwner does not know both U and T [2], this is impossible.
31 //
32 // [1] There is no way of passing static type information across concrete derived implementation classes, when only the type-erased base is known.
33 // shared_ptr doesn't need this extraction, because it has a direct pointer; however CopiedPtr creates a *new* pointer that is unknown.
34 // [2] A CopiedPtr<Base> that is constructed from CopiedPtr<Derived> has no type information about the dynamic type held by the latter.
35 // The dynamic type needn't be Derived, also VeryDerived (which inherits Derived) is possible -- through a previous CopiedPtr<Derived> constructor call
36 // to either CopiedPtr(VeryDerived*), CopiedPtr(Derived*) or CopiedPtr(const CopiedPtr<VeryDerived>&). Note that in the 2nd case, the CopiedPtr has never
37 // had any static type information, the user upcast the pointer already before passing it to the constructor. But even in the 1st and 3rd case, the
38 // static type information is type-erased and only available within a derived object like PtrOwner.
39 //
40 // Conclusion: In the general case, it is not possible to maintain a single type-erased object for the implementation of derived-to-base conversions.
41 // Some special cases could be optimized using RTTI trial and error, but that's just a hack which doesn't solve the inherent underlying problem.
42 // The current implementation is very clean: No static_cast, dynamic_cast or typeid is used.

78  // Default pointer owner
79  template <typename T, typename U, typename C, typename D>
80  struct PtrOwner : PtrOwnerBase<T>
81  {
82  PtrOwner(U* pointer, C cloner, D deleter, bool doClone = false)
83  : pointer(pointer)
84  , cloner(cloner)
85  , deleter(deleter)
86  {
87  assert(pointer);
88 
89  // Exception safety: If cloning fails, constructor will be aborted, reverting surrounding new operator
90  if (doClone)
91  this->pointer = cloner(pointer);
92  }
93 
94  virtual ~PtrOwner()
95  {
96  if (pointer)
97  deleter(pointer);
98  }
99 
100  virtual PtrOwner* clone() const
101  {
102  return new PtrOwner(pointer, cloner, deleter, true);
103  }
104 
105  virtual T* getPointer() const
106  {
107  return pointer;
108  }
109 
110  U* pointer;
111  C cloner;
112  D deleter;
113  };

149  // Used for U* -> T* derived-to-base conversion
150  // See notes at the beginning of the document
151  template <typename T, typename U>
152  struct PtrIndirection : PtrOwnerBase<T>
153  {

155  : base(originBase->clone())

158 
159  explicit PtrIndirection(PtrOwnerBase<U>* sourceBase, MoveTag)
160  : base(sourceBase)

162  }
163 
164  virtual ~PtrIndirection()
165  {
166  delete base;
167  }
168 
169  virtual PtrIndirection<T, U>* clone() const
170  {
171  return new PtrIndirection<T, U>(base, CopyTag());
172  }
173 
174  virtual T* getPointer() const

176  return base->getPointer();
